# ASU Projects — Overview

This repository collects coursework, assignments, projects, and related data used while pursuing a Master's degree at Arizona State University. Each top-level folder corresponds to a course assignment, project, or utility; short descriptions are provided below to help you navigate the repository.

Below is a concise summary of each top-level folder and what you will find inside.

- `cse-572-hw1` : Homework for CSE 572 — data science notebooks and supporting files. Contains Titanic survival prediction notebooks, `requirements.txt`, and the `data/` folder with train/test CSVs.
- `cse-572-hw3` : Additional assignments for CSE 572. Includes clustering/recommender notebooks, helper scripts, and results subfolders for experiments and outputs.
- `cse511-assignment2` : Assignment 2 for CSE 511. Provides a template Scala project under `assignment2-template/` and sample data files used by the assignment.
- `cse511-assignment3` : Assignment 3 for CSE 511. SBT-based Scala/Spark project with source code in `src/`, build files, and artifacts in `target/`.
- `cse511-assignment4` : Assignment 4 for CSE 511. Python-based assignment scripts, helper/test files and JSON fixtures used by the grader/tester.
- `cse511-Project-1` : Project work for CSE 511. Contains Docker config, data, scripts, and a simple interface and tester for the project.
- `CSE546-FALL-2025` : Work for CSE 546 (Fall 2025). Contains an app-tier and web-tier, face-recognition notebooks/scripts, model artifacts (`data.pt`) and dataset examples.
- `cse571-hw1` : Homework for CSE 571. ROS package layout and launch/configuration files, tests, scripts, and supporting resources.
- `cse572-project` : Project work for CSE 572. Notebooks and code for loan prediction experiments, artifacts, and data resources.

Other common files:

- `.gitignore` : Repository-level ignore rules for environments, build artifacts, datasets, and temporary files.

How to use this repository

- Browse the folder for the course or assignment you need. Each course folder typically contains `README.md` (when present), notebooks, scripts, and a `data/` directory for small example datasets.
